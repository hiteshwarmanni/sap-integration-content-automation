# Error Capture Fix for Transport Logs

## Issue Identified
Error details were not being captured in the database transport logs when transport operations failed.

## Root Cause
The `TRANSPORT_LOGS` table was missing the necessary columns to store error information:
- `errorMessage` - The main error message
- `errorDetails` - JSON-formatted detailed error information
- `failedStep` - Which step in the transport process failed
- `errorStackTrace` - Full stack trace for debugging

## Databases Affected
1. **SQLite** (local development) - Missing error columns
2. **HANA** (Cloud Foundry deployment) - Missing error columns in table definition

## Fix Applied

### 1. HANA Database Table Definition
Updated `db/src/TRANSPORT_LOGS.hdbtable` to include error columns:
```sql
COLUMN TABLE "TRANSPORT_LOGS" (
    "ID" INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    "PROJECT_NAME" NVARCHAR(255),
    "ENVIRONMENT" NVARCHAR(100),
    "USER_NAME" NVARCHAR(255),
    "TIMESTAMP" NVARCHAR(50),
    "STATUS" NVARCHAR(50),
    "SOURCE_PACKAGE_ID" NVARCHAR(255),
    "TARGET_PACKAGE_ID" NVARCHAR(255),
    "SOURCE_IFLOW_ID" NVARCHAR(255),
    "TARGET_IFLOW_ID" NVARCHAR(255),
    "SOURCE_IFLOW_NAME" NVARCHAR(255),
    "TARGET_IFLOW_NAME" NVARCHAR(255),
    "TIME_TAKEN_SECONDS" INTEGER,
    "ERROR_MESSAGE" NCLOB,           -- Added
    "ERROR_DETAILS" NCLOB,            -- Added
    "FAILED_STEP" NVARCHAR(100),      -- Added
    "ERROR_STACK_TRACE" NCLOB         -- Added
)
```

### 2. SQLite Database Schema
Added the following columns to the existing `transport_logs` table:
```sql
ALTER TABLE transport_logs ADD COLUMN errorMessage TEXT;
ALTER TABLE transport_logs ADD COLUMN errorDetails TEXT;
ALTER TABLE transport_logs ADD COLUMN failedStep TEXT;
ALTER TABLE transport_logs ADD COLUMN errorStackTrace TEXT;
```

## Verification Steps

### For Local Development (SQLite)
1. Check the schema:
   ```bash
   sqlite3 logs.db ".schema transport_logs"
   ```
2. Verify columns exist: `errorMessage`, `errorDetails`, `failedStep`, `errorStackTrace`

### For Cloud Foundry (HANA)
1. Deploy the updated `db/src/TRANSPORT_LOGS.hdbtable` to HANA
2. The HANA deployment will create the table with the new columns
3. Existing records will have NULL values for the error columns

## Impact
- **No breaking changes** - Existing successful transport logs will have NULL error fields
- **Enhanced debugging** - Failed transports will now capture complete error details including:
  - What step failed (Authentication, CSRF Token, Download Zip, Edit Zip, Upload iFlow)
  - HTTP status codes from API failures
  - Complete API error responses
  - Full stack traces for debugging

## Testing
To test the error capture:
1. Trigger a transport operation that will fail (e.g., invalid credentials)
2. Check the `transport_logs` table for the failed entry
3. Verify that `errorMessage`, `errorDetails`, `failedStep`, and `errorStackTrace` contain the expected error information

## Deployment Notes
- For SQLite environments: The columns have been manually added
- For HANA environments: Deploy the updated table definition file to apply the schema changes
- The auto-migration code in `server/db.js` will handle future fresh installations
